gitの基本ワークフロー
1.[.git]というローカルリポジトリを作成
  git init
2.ファイルの変更をステージングエリアへ追加する。
  git add
3.ローカルリポジトリにコミットする。
  git commit ファイル名
4.リモートリポジトリにプッシュする。
  git push -v ファイル名

5.現在の状況を確認する。
  git status

6.変更履歴を確認しよう。
  git logコマンド
  →ローカルリポジトリにコミットされたログを確認する。

  ・git logコマンドのオプション
    --oneline     : 1行で表示する
    -p ファイル名 : ファイルの差分を表示する
    -n 数値       : 表示するログ数を指定できる
